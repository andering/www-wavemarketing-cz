# Mobile Responsive Refinements Plan

## Goal

Implement the requested mobile-only refinements for the WAVE Marketing homepage while preserving the docs-first workflow and desktop behavior:

- Apply the Stitch-inspired hero background treatment on mobile.
- Hide the hero picture on mobile.
- Hide the `Kdo jsme` decorative wave on mobile only.
- Replace the mobile `Menu` text trigger with a hamburger control.
- Record in the roadmap that responsive work must track mobile-vs-desktop responsibility differences, not only spacing.

## Assumptions

- “Remove the picture on mobile” means visually hide the hero image at the mobile breakpoint. The image may still be present in markup and may still download because the desktop still uses the approved local asset.
- The Stitch reference `projects/1618198505265661547/screens/ecf1dc19c6614df3b071e2be793696d0` should be treated as visual direction only. Do not copy Stitch-hosted assets, copy, or remote URLs.
- “Mobile” should follow the existing component-owned breakpoints unless the implementation reveals a strong reason to consolidate them: hero at `max-width: 900px`, intro at `max-width: 840px`, header at `max-width: 720px`.

## Implementation Steps

1. Update source-of-truth docs.
   - In `DESIGN.md`, document a mobile hero exception: desktop keeps the approved organic image frame, while mobile uses a warm sand surface with blurred ambient teal and warm-brown/cream blobs inspired by Stitch node `ecf1dc19c6614df3b071e2be793696d0`.
   - In `DESIGN.md`, document that mobile navigation uses a hamburger trigger that opens the existing right-side offcanvas panel.
   - In `DESIGN.md`, document that the `intro-statement` decorative wave panel is desktop/tablet only and is omitted on mobile to reduce visual weight.
   - In `page-map.md`, add explicit desktop/mobile responsibilities for `site-header`, `marketing-hero`, and `intro-statement`.
   - In `website-production-roadmap.md`, add the requested decision: responsive refinement must track differences in section/component responsibility between mobile and desktop, not just breakpoint spacing.

2. Add lightweight invariants.
   - Extend `src/tests/site-content.test.ts` to check that the docs record the mobile hero Stitch reference, mobile hamburger trigger, mobile intro wave omission, and roadmap responsive-responsibility note.
   - Keep tests textual and source-of-truth oriented; avoid visual snapshots.

3. Implement the mobile hero treatment in `MarketingHero.astro`.
   - Keep desktop styles unchanged.
   - In the existing `@media (max-width: 900px)` block, hide `.hero__visual` with `display: none`.
   - Add a mobile-only hero background using local CSS gradients/pseudo-elements, based on the reference: warm base surface, soft teal ambient blob, and warm secondary/cream blob. Use existing design tokens where possible and avoid external assets.
   - Ensure hero copy and CTAs remain readable above the decorative background.

4. Implement the hamburger trigger in `SiteHeader.astro`.
   - Replace the visible `Menu` text inside `.site-header__mobile-nav` with a hamburger icon built from markup/CSS or CSS pseudo-elements.
   - Preserve the checkbox/label offcanvas mechanism.
   - Keep an accessible text label through `aria-label` and/or visually hidden text.
   - Preserve focus visibility and maintain a practical touch target.

5. Remove the `Kdo jsme` wave on mobile in `IntroStatement.astro`.
   - Keep desktop/tablet SVG wave markup and styling unchanged.
   - In the existing `@media (max-width: 840px)` block, hide `.intro__visual` with `display: none`.
   - Keep the text stack, lead typography, and accent divider visible.

## Verification

Run:

```bash
npm run test
npm run build
```

Browser-check responsive behavior after implementation:

- At mobile widths around `375px`: hero has the Stitch-inspired background, hero image is absent, hamburger is visible, offcanvas opens from the right, and `Kdo jsme` wave is absent.
- Around `720px`, `840px`, and `900px`: confirm component breakpoint transitions are intentional and not visually broken.
- At desktop width: hero image returns, intro wave returns, centered split navigation remains unchanged.

## Risks

- CSS-only image hiding is visually correct but does not guarantee the hero image is not downloaded on mobile. If mobile network performance is the goal, change the implementation to conditionally serve the image with responsive markup in a follow-up.
- The hero, intro, and header currently use different breakpoints. The plan preserves existing ownership to minimize change, but visual review should confirm the transitions feel coherent.
